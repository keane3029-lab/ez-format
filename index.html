The .ez format

A key-value data format built to be readable by anyone, on any machine, without needing to trust the person who sent the file. This site documents the format, the reference tools, and how to set it up.

Overview

.ez files store simple key-value data — think of it as a plain-text alternative to JSON, readable line by line without any nesting or punctuation to memorize.

Three things it's built around:

  • Simple — one line, one piece of data, always KEY VALUE.
  • Data-only — a .ez file describes things. It never executes commands on the machine that opens it. See the Security page.
  • Double-click friendly — set up once, then .ez files open like any normal file type.

Quick start

1
Write a .ez file — it's just text, any editor works:
hello.ez
meta name Hello World >
status locked_in >
2
Run it from a terminal:
shell
python3 ez_run.py hello.ez
3
Or set up double-click support once — see Double-click setup — and just open the file normally from then on.
