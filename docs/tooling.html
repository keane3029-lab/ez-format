Runner & parser

The two reference implementations that ship with the format.

The runner — ez_run.py

The reference runner does exactly one job: it reads the file, and if it contains @anim, it opens a URL in the system's default browser. That's the entire behavior — it doesn't read other keys, doesn't touch other files, doesn't do anything conditional based on the data.

shell
python3 ez_run.py somefile.ez

The parser — infect.js

A small reference implementation for reading .ez data into a plain object, in JavaScript:

javascript
import infect from './infect.js';
import fs from 'fs';

const content = fs.readFileSync('welcome.ez', 'utf8');
const data = infect(content);
console.log(data);

It splits on >, then splits each entry on the first space only — so multi-word values come through intact instead of getting truncated.
